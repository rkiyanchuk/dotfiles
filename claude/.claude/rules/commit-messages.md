# Commit messages

Use **scope-prefixed** subjects (Linux/Git/Go style), never Conventional Commits
(`feat:`/`fix:`/`chore:`):

```sh
scope: imperative description
```

Scope = the natural area touched (subsystem, package, module, dir). Omit only
when a change is genuinely cross-cutting.

- Subject ≤ **50 chars**, imperative, capitalized, no trailing period.
- One blank line, then body (for non-trivial changes); wrap body at **72**.
- Body explains **what** and **why**, not how.

## Trailers

`Key: value` lines at the end, after a blank line (`git interpret-trailers`):
`Signed-off-by`, `Co-authored-by`, `Reviewed-by`, `Tested-by`, `Reported-by`,
`Fixes:`/`Closes: #123`, `Link:`.

Claude Code attribution — by who wrote the **code**, not the message:

- Code generated by Claude Code → add
  `Co-authored-by: Claude Code <noreply@anthropic.com>`.
- Claude only wrote the message (human wrote the code) → no attribution.
